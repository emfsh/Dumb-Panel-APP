## 更新内容

### 兼容呆呆面板 v2.2.18 / v2.2.19

- **任务默认 Python 版本对齐面板**：新建任务时，Python 版本不再固定为 `3.12`，会自动跟随面板依赖管理里的默认 Python 版本；编辑已有任务时仍保持任务原本保存的版本。
- **系统更新提示适配新更新链路**：系统设置页现在会区分 Watchtower 托管更新、Docker 镜像更新和二进制后台更新，不再统一提示成“面板将自动重启”。
- **系统更新状态补齐**：APP 端新增轻量更新状态展示，可查看当前更新状态、阶段和后端返回的状态说明，减少“点了更新但不知道现在在做什么”的情况。
- **仪表盘资源状态修正**：当面板当前环境拿不到内存总量时，APP 不再把内存状态误显示成 `0%`，会明确提示资源采集不可用。

### 优化

- **Python 运行时信息复用**：任务页和依赖页共用同一套 Python 运行时结构，减少两边字段解析分叉带来的后续适配风险。
- **版本说明同步**：README 里的兼容面板版本和适配重点已同步到 `v2.2.19`。

### 说明

- 本次更新主要聚焦 APP 与面板 `v2.2.18 / v2.2.19` 的关键行为适配，没有引入新的数据库结构或移动端大规模页面重构。
